About me

J2EE

My professional programming education started with Java and J2EE. I worked on various enterprise web application architectures - mostly three-tier designs, with object relational mappers on top of relational data stores and some more or less common view technology. Later, I built message driven systems and more asynchronous designs deployed to small clusters.
I implemented persistence layers on top of JPA or MongoDB, caching layers and a high-performance library for pub/sub pattern (called mbassador).

Expand the box

With more practice, I started moderating code reviews and helped to educate bachelor students in their programming efforts. I spent time with JVM profiling learning about garbage collection, heap organization and JIT optimizations.
Writing code for multi-threaded environments is another (ongoing) challenge that made me learn about the difficulties of code synchronization and the value of immutable data [structures].
I moved from Java to other JVM based languages like Groovy and Scala. Recently a lot of my programming moved to node.js using JavaScript and coffeescript. I built mailody and kermit.

Domain Modeling

My interest for Domain Specific Languages started in a university class about meta-modeling. I learned Xtext and for my diploma thesis I implemented a small model-to-model transformation language inspired by Xpath but with smaller syntax and Java integration. The language was used to implement JDO to JPA persistence model migrations of a larger code base. I was also inspired by Eric Evans ideas about Domain Driven Design and the ubiquitous language. I strongly believe that a good fit of the domain model and the real world is a huge win. Concise, natural, well-tested and locally scoped domain models provide a solid building block for large software landscapes.
